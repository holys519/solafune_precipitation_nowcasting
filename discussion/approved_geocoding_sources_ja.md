# 【公式】承認済みジオコーディングソース：許可リストと追加申請

以下は、Solafune運営による公式告知の日本語訳です。

## 背景

地名を地理座標（EPSG:4326 / WGS 84）へ変換することは、外部データの利用ではなく、
**特徴量変換**として扱われます。ただし、次の条件を満たす必要があります。

- 公式の承認リストに掲載されたジオコーディングソースだけを使用すること
- 提出時に使用したソースを明確に記載すること
- 検証時に運営側が同じ変換を再現できること

## 承認済みソース

現在の承認済みジオコーディングソースは次の2つです。

- **GeoNames**
- **Nominatim（OpenStreetMap）**

## 追加の明確化

無料であっても登録が必要なソースは、無料枠だけで変換を再現できる場合に使用できます。

リクエスト回数に制限があるソースも、速度ではなく**再現可能性**を満たすことが条件です。

有料サービスは使用できません。

## 許可される変換の範囲

この変換例外で許可されるのは、地名から取得した位置と時刻から、外部データセットなしに
閉形式で計算できる特徴量だけです。

### 許可される例

- 緯度・経度
- 北半球・南半球
- 緯度・経度の三角関数エンコーディング（sin/cos）
- 太陽ジオメトリ
- local solar time（現地太陽時）

### 許可されない例

次の特徴量は、外部データセットとの結合が必要になるため、ライセンスが無料・公開であっても
使用できません。

- 標高
- 海岸線までの距離
- 気候区分

## 新しいソースの申請

承認リストにないソースを使用したい場合は、次の情報を添えて公式トピックへ返信します。

- ソース名
- ソースへのリンク
- ライセンスまたは利用規約
- 運営側で検索を再現する方法

運営が確認した後、承認リストが更新されます。

## 本プロジェクトでの実装方針

ジオコーディングを実装する場合は、APIを学習・推論のたびに呼び出すのではなく、結果を固定
したテーブルとして保存します。

保存すべき項目は次のとおりです。

- 元の`name_location`
- 使用ソース（GeoNamesまたはNominatim）
- ソースのレコードIDまたは検索URL
- canonical name
- latitude / longitude
- 座標系がEPSG:4326 / WGS 84であること
- 取得日時
- 変換スクリプト

この情報を提出時に文書化すれば、運営側の検証時に同じ変換を再現できます。

## 実験に利用できる特徴量

現在の`exp018`には位置・太陽特徴が入っていないため、次の特徴量は規約に沿った追加候補です。

- `latitude / 90`
- `sin(longitude)`, `cos(longitude)`
- `sin/cos(day_of_year)`
- local solar timeのsin/cos
- 太陽高度または太陽天頂角
- 北半球・南半球フラグ

ただし、地名のone-hotや、地域ごとの平均降水量・気候情報を作ることは避けます。学習地域と
評価地域は重複していないため、地域IDの暗記ではなく、位置・時刻に基づく物理的な特徴として
利用する必要があります。
