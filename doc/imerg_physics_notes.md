## 1. 目的変数 = IMERG はどう作られているか

IMERG (Integrated Multi-satellitE Retrievals for GPM) の30分×0.1°格子は、
単一センサーの観測ではなく**多段の合成プロダクト**である:

1. **PMW (受動マイクロ波) 直接観測**: GMI/SSMIS/AMSR2/MHS/ATMS等の低軌道コンステレー
   ションが各地点を**不定期 (平均1〜3時間おき)** に通過し、そのスワス内では物理的に
   最も確かな降水推定を与える。
2. **モーフィング (MORPH)**: PMW通過の合間は、直前 (と直後 — Finalランは双方向) の
   PMW推定を**移流ベクトルで空間移動**させて補間する。移流ベクトルはV06以降、雲IRでは
   なく**再解析 (MERRA-2/GEOS FP) の全可降水量の動き**から作られる。
3. **IRフィル**: PMWが古すぎる場合、静止衛星IRからの推定 (PERSIANN-CCS系) を
   Kalmanフィルタ重みで混合する。
4. **ゲージ較正**: Finalランは月単位で雨量計に較正される (30分構造はほぼ不変)。

### ここから導かれる、既に実証済みの法則

| 法則 | 我々の実測 |
| --- | --- |
| モーフィングにより時間方向に強い平滑構造 | lag1自己相関0.561、予測の時間平滑化でLB −0.0045 |
| 双方向モーフィング+IRフィルで未来側にも情報 | 平滑化の最適重みが next:prev = 3:2〜3:1 |
| 合成プロダクトゆえ実効解像度は0.1°より粗い | パワーの81%が波数1-2、blur σ0.5-1が微益 |
| 0.01量子化 | 実在するが丸めは**効果ゼロ** (クローズ済み) |

### 未検証の法則 (→ g_eda/exp005 E-7)

**H1: フレームは「PMWフレッシュ」と「モーフ」の2状態を持つ**
- モーフフレームは前フレームの移流でほぼ説明でき (イノベーション小)、フレッシュ
  フレームで場が「ジャンプ」する。イノベーション (1 − 隣接フレーム相関、または
  移流残差) の分布は**二峰性**になるはず。
- 高イノベーションフレームの時間間隔はPMW再訪間隔 (~1-3h) に対応する準周期性を
  持つはず。
- **使い道**: (a) フレーム状態で時間平滑化の重みを適応化 (モーフフレームは近傍への
  依存を強めてよい)。(b) 検証時に「フレッシュフレームだけで測るtile_rmse」を見れば、
  モデルがPMW情報 (=IRから原理的に予測しにくい成分) をどれだけ外しているかが分かる。
- **リスク**: eval側でフレーム状態を知る術は予測残差しかない → 入力IRとの不整合度で
  代理できるか、が検証ポイント。

**H2: GPM場の時間発展は「移流」でよく説明できる**
- 純persistenceのスキルは0.315 (実測済み) だが、**移流補正付きpersistence**
  (前フレームをphase correlationシフトしてから比較) はもっと高いはず。IMERG自身が
  移流で作られているため。
- **使い道**: 予測の時間平滑化を「移流付き平滑化」へ拡張 (隣接フレーム予測をシフト
  してから混合)。現在の静的平滑化が−0.0045なら、移流版はその上位互換になり得る。
  また、Round 6モデルへの「GPM場の自己移流」構造の組込み根拠になる。
- **注意**: IR雲の光学フローは無価値と実測済み (ρ≈−0.05) だが、それは**雲とGPM移流
  ベクトルが別物**だからで、GPM場自身の移流とは別の話。

## 2. 入力 = 静止衛星IRの物理

**H3: 視差 (パララックス) は決定論的な幾何で、evalにも計算できる**
- 静止衛星は赤道上の1点 (GOES-East 75.2°W / Himawari 140.7°E / Meteosat 0°) から
  見るため、高い雲は**衛星直下点から遠ざかる方向**へ (雲高 × tan(視角)) だけずれて
  写る。方向は地点座標と直下点から**閉形式で計算できる** (許可された特徴変換の範囲)。
- 実測 (`outputs/g_eda/exp001/parallax_shift_by_location.csv`) では地点ごとの
  経験シフトと相関ゲイン (最大+0.089) が既にある。**幾何予測と経験値が整合するなら、
  真値が無くて測れないeval地点にも登録補正を外挿できる** — これがG-017 (exp013a) を
  eval側で成立させる鍵。
- → g_eda/exp005 E-8で幾何モデルを検証し、eval地点の予測シフト表を出力する。

**H4: IRの盲点は系統的**
- IR窓は雲頂温度しか見えない: (a) **warm rain** (背の低い雨、海岸/海洋性) は冷たく
  写らず見逃す、(b) **cirrus** (高い巻雲) は冷たいが降らない偽警報。PMWフレッシュ
  フレームとIRベース入力の不整合が大きいのはこの2状況のはず。
- Meteosat圏 (アフリカ/欧州) のwet IoU劣化 (0.186) の一部はこれかもしれない
  (regime差はE-6でhimawariと層別比較)。
- **使い道**: 不整合パターンの層別が分かれば、occurrence headの衛星別しきい値の
  さらに先 (例: 特定regime限定のcalibration) に根拠を与える。

**H5: 太陽ジオメトリ・現地時刻は閉形式で許可されている**
- 公式ルーリング: **緯度経度・sin/cosエンコード・太陽ジオメトリ・local solar time
  は許可**。標高・海岸線距離・気候区分・海陸率は外部データで**禁止**。
- 降水には強い日周期がある (陸: 午後対流ピーク / 海: 早朝ピーク)。現在のday/night
  フラグ (VIS輝度) より情報量の多い **local solar hourのsin/cos** を入力に足せる。
- → E-9: train GPMで地点×現地時刻の降水気候を実測し、日周期の振幅がノイズを超える
  かを確認してから Round 6 の入力候補にする。
- 座標の取得は**GeoNames/Nominatimのみ許可** (GeoTIFFの座標タグは運営が意図的に
  除去済み — 恒等行列を確認)。分析用の近似座標と、提出用の正規手続きを区別すること。

## 3. タイル設計から

**H6: タイルはevent-centered** (確認済み、center/edge比1.07は無視可)。
train/evalの空間重複 (3ペア) はexp014系で回収済み。**時間方向の重複**
(同一時刻の隣接タイル間で境界が繋がる) は north_sumatra/aceh ペアで既に活用
しているが、eval地点同士の隣接 (mekong_delta/quang_nam等) で予測の**空間整合**
を強制する後処理は未検討 — 優先度低 (境界画素のみ)。
